# Git 简介

## 什么是 Git？

**Git** 是一个开源的**分布式版本控制系统**，由 Linus Torvalds（Linux 之父）于 2005 年创建。

### 版本控制是什么？

想象一下这个场景：

你在写一篇重要的文档，为了防止出错，你可能会这样保存文件：

```
我的项目_v1.doc
我的项目_v2.doc
我的项目_v2_最终版.doc
我的项目_v2_最终版_真的最终版.doc
我的项目_v2_最终版_真的最终版_这次是真的.doc
```

这种方式虽然能工作，但问题很多：

- ❌ 文件名混乱，难以管理
- ❌ 占用大量存储空间
- ❌ 难以知道每个版本改了什么
- ❌ 多人协作时容易冲突
- ❌ 很难回到之前的某个版本

**版本控制系统**就是用来解决这些问题的工具。它可以：

- ✅ 自动记录每次修改
- ✅ 查看任何时间点的文件状态
- ✅ 对比不同版本的差异
- ✅ 多人协作不会互相覆盖
- ✅ 轻松回退到任何历史版本

## 为什么选择 Git？

### 1. 分布式架构

传统的版本控制系统（如 SVN）是**集中式**的：

```
集中式 (SVN):
    中央服务器
    /    |    \
  用户A 用户B 用户C
```

Git 是**分布式**的：

```
分布式 (Git):
  用户A ←→ 用户B
    ↕        ↕
  中央服务器 ← 用户C
```

**优势**：
- 每个用户都有完整的代码历史
- 不依赖网络也能工作
- 速度更快
- 更安全，不怕中央服务器损坏

### 2. 强大的分支功能

Git 的分支管理极其轻量和高效：

```bash
# 创建新分支只需 1 秒
git branch feature-new-login

# 切换分支也只需 1 秒
git checkout feature-new-login
```

这让你可以：
- 🔄 轻松尝试新想法，不影响主代码
- 🤝 多人并行开发不同功能
- 🐛 快速修复紧急 bug
- 🧪 安全地进行实验

### 3. 完整的历史记录

Git 记录了项目的完整历史：

```bash
# 查看历史
git log

# 查看某个文件的修改历史
git log -- README.md

# 查看谁修改了某一行代码
git blame app.js
```

### 4. 强大的社区和生态

- **GitHub** - 全球最大的代码托管平台，7000万+ 开发者
- **GitLab** - 企业级 DevOps 平台
- **Gitee** - 国内领先的代码托管平台
- 无数集成工具和服务

## Git 的核心概念

在开始使用 Git 之前，理解这些核心概念很重要：

### 1. 仓库（Repository）

**仓库**（简称 repo）是 Git 用来存储项目代码和历史记录的地方。

```
我的项目/
├── .git/          # Git 仓库（存储所有历史记录）
├── index.html     # 你的项目文件
├── style.css
└── script.js
```

### 2. 提交（Commit）

**提交**是项目的一个快照。每次提交都会记录：

- 📝 修改了哪些文件
- 👤 谁做的修改
- ⏰ 什么时间修改的
- 💬 为什么修改（提交信息）

```bash
# 创建一个提交
git commit -m "添加用户登录功能"
```

每个提交都有一个唯一的 ID（SHA-1 哈希值）：

```
commit 3a7d2f9c8b1e4a6d5c9f8e7a6b5c4d3e2f1a0b9c
```

### 3. 分支（Branch）

**分支**让你可以在不影响主代码的情况下开发新功能。

```
主分支 (main):    A --- B --- C --- D
                           \
新功能分支:                  E --- F
```

### 4. 工作区、暂存区和仓库

Git 有三个重要的区域：

```
工作区              暂存区              仓库
(Working Directory) (Staging Area)     (Repository)

修改文件    →    git add    →    git commit
              准备提交           正式提交
```

## Git 的工作流程

一个典型的 Git 工作流程：

```bash
# 1. 修改文件
vim index.html

# 2. 查看修改
git status

# 3. 添加到暂存区
git add index.html

# 4. 提交到仓库
git commit -m "更新首页标题"

# 5. 推送到远程仓库
git push
```

## Git vs GitHub

很多初学者会混淆这两个概念：

| Git | GitHub |
|-----|--------|
| 版本控制**软件** | 代码托管**网站** |
| 安装在本地 | 运行在云端 |
| 命令行工具 | Web 界面 + Git 功能 |
| 开源免费 | 免费 + 付费服务 |
| 1000+ 命令 | 社交功能、CI/CD 等 |

{{< hint info >}}
**简单来说**：
- **Git** = 版本控制工具（软件）
- **GitHub** = 使用 Git 的代码托管平台（网站）
{{< /hint >}}

类似的平台还有：
- **GitLab** - 功能强大的 DevOps 平台
- **Gitee** - 国内的代码托管平台
- **Bitbucket** - Atlassian 的代码托管服务

## Git 的应用场景

Git 不仅适用于代码，还可以管理：

- 💻 **软件开发** - 最主要的应用场景
- 📚 **文档编写** - Markdown、LaTeX 等
- 🎨 **设计稿** - SVG、配置文件
- 🔧 **配置管理** - 服务器配置、dotfiles
- 📖 **书籍写作** - 很多技术书籍用 Git 管理
- 🎓 **学术论文** - 版本控制和协作

## Git 的优势总结

| 优势 | 说明 |
|------|------|
| 🚀 **速度快** | 大部分操作在本地完成，无需网络 |
| 💪 **功能强大** | 支持复杂的分支和合并操作 |
| 📦 **存储高效** | 智能压缩，节省空间 |
| 🔒 **数据完整性** | 使用 SHA-1 保证数据不被篡改 |
| 🌍 **分布式** | 每个开发者都有完整历史 |
| 🆓 **开源免费** | 完全免费，社区活跃 |
| 🏆 **业界标准** | 被全球数千万开发者使用 |

## 💡 练习题

完成以下练习，检验你的理解：

{{< expand "练习 1：概念理解" >}}
**问题**：以下哪些说法是正确的？

A. Git 是一个集中式版本控制系统
B. Git 和 GitHub 是同一个东西
C. 每个 Git 用户都有完整的代码历史
D. Git 只能用于管理代码

{{< expand "查看答案" >}}
**答案**：C

**解析**：
- A 错误：Git 是**分布式**版本控制系统
- B 错误：Git 是软件，GitHub 是网站平台
- C **正确**：这是分布式架构的特点
- D 错误：Git 可以管理任何文本文件
{{< /expand >}}
{{< /expand >}}

{{< expand "练习 2：场景应用" >}}
**问题**：你正在开发一个网站项目，需要添加一个新功能，但不想影响当前正常运行的代码。你应该使用 Git 的什么功能？

A. 提交（Commit）
B. 分支（Branch）
C. 暂存区（Staging Area）
D. 仓库（Repository）

{{< expand "查看答案" >}}
**答案**：B

**解析**：
使用**分支**可以让你在不影响主代码的情况下开发新功能。你可以创建一个新分支，在上面开发和测试新功能，确认没问题后再合并回主分支。
{{< /expand >}}
{{< /expand >}}

{{< expand "练习 3：思考题" >}}
**问题**：相比传统的"复制粘贴"方式保存不同版本的文件，Git 有哪些优势？请列举至少 3 个。

{{< expand "参考答案" >}}
**参考答案**：

1. **节省空间**：Git 使用智能算法只存储文件差异，而不是完整复制
2. **清晰的历史记录**：每次修改都有明确的说明、时间和作者信息
3. **轻松回退**：可以快速回到任何历史版本
4. **多人协作**：可以合并多人的修改，自动处理冲突
5. **分支管理**：可以同时开发多个功能而不互相影响
6. **安全性**：有备份机制，不容易丢失代码
