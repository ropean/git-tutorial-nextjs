# Git 完全指南

{{< hint info >}}
**欢迎来到 Git 完全指南！** 这是一套全面系统的 Git 教程，帮助你从零基础到精通 Git 版本控制系统。
{{< /hint >}}

## 为什么学习 Git？

Git 是目前世界上最流行的分布式版本控制系统，被数百万开发者使用。无论你是：

- **编程新手** - 学习如何管理和追踪代码变化
- **专业开发者** - 掌握团队协作和代码管理的最佳实践
- **开源贡献者** - 了解如何参与开源项目
- **技术团队负责人** - 建立高效的代码协作流程

Git 都是你必须掌握的核心技能。

## 学习路径

本教程采用**渐进式学习**方式，分为以下几个阶段：

### 🚀 第一阶段：快速开始（入门必读）

适合完全没有接触过 Git 的初学者，30分钟快速上手。

- [Git 简介](docs/getting-started/introduction/) - 什么是 Git，为什么要用 Git
- [安装配置](docs/getting-started/installation/) - 在不同平台上安装和配置 Git
- [第一个仓库](docs/getting-started/first-repository/) - 创建你的第一个 Git 项目
- [基本工作流程](docs/getting-started/basic-workflow/) - 了解 Git 的日常使用流程

### 📚 第二阶段：基础操作（核心知识）

掌握 Git 的核心概念和常用命令。

- [仓库初始化](docs/basics/init-clone/) - 创建和克隆仓库
- [文件操作](docs/basics/add-commit/) - 添加、提交文件
- [查看状态和历史](docs/basics/status-log/) - 了解项目状态
- [比较差异](docs/basics/diff/) - 查看文件变化
- [撤销更改](docs/basics/undo/) - 撤销和重置操作
- [忽略文件](docs/basics/gitignore/) - 配置 .gitignore

### 🌿 第三阶段：分支管理（团队协作基础）

理解 Git 最强大的功能之一：分支管理。

- [分支概念](docs/branching/concepts/) - 理解分支的工作原理
- [创建和切换分支](docs/branching/create-switch/) - 分支的基本操作
- [合并分支](docs/branching/merge/) - 整合不同分支的代码
- [解决冲突](docs/branching/conflicts/) - 处理合并冲突
- [变基操作](docs/branching/rebase/) - 使用 rebase 整理提交历史
- [分支策略](docs/branching/strategies/) - Git Flow、GitHub Flow 等工作流

### 🔄 第四阶段：远程协作（多人开发）

学习如何与团队成员协作，使用远程仓库。

- [远程仓库概念](docs/remote/concepts/) - 理解远程仓库
- [克隆与推送](docs/remote/clone-push/) - 获取和上传代码
- [拉取与获取](docs/remote/pull-fetch/) - 同步远程更新
- [远程分支](docs/remote/branches/) - 管理远程分支
- [Fork 与 Pull Request](docs/remote/fork-pr/) - 参与开源项目
- [常见平台](docs/remote/platforms/) - GitHub、GitLab、Gitee 使用指南

### 💡 第五阶段：进阶技巧（提升效率）

深入学习 Git 的高级功能和技巧。

- [标签管理](docs/advanced/tags/) - 版本发布和标记
- [储藏更改](docs/advanced/stash/) - 临时保存工作进度
- [子模块](docs/advanced/submodules/) - 管理项目依赖
- [钩子](docs/advanced/hooks/) - 自动化工作流
- [别名配置](docs/advanced/aliases/) - 提升命令效率
- [高级搜索](docs/advanced/search/) - 查找代码和历史
- [重写历史](docs/advanced/rewrite-history/) - 修改提交记录
- [性能优化](docs/advanced/performance/) - 大型仓库优化

### 🎯 第六阶段：最佳实践（专业技能）

学习业界广泛认可的最佳实践和规范。

- [提交规范](docs/best-practices/commit-messages/) - 编写优秀的提交信息
- [分支命名](docs/best-practices/branch-naming/) - 规范的分支命名
- [代码审查](docs/best-practices/code-review/) - Pull Request 最佳实践
- [团队协作](docs/best-practices/team-workflow/) - 团队工作流程
- [安全实践](docs/best-practices/security/) - 避免敏感信息泄露
- [持续集成](docs/best-practices/ci-cd/) - Git 与 CI/CD 集成

### 🔧 第七阶段：问题排查（实战应用）

解决 Git 使用过程中遇到的常见问题。

- [常见错误](docs/troubleshooting/common-errors/) - 错误信息和解决方案
- [恢复数据](docs/troubleshooting/recovery/) - 找回丢失的代码
- [清理仓库](docs/troubleshooting/cleanup/) - 清理大文件和历史
- [迁移仓库](docs/troubleshooting/migration/) - 仓库迁移指南
- [FAQ](docs/troubleshooting/faq/) - 常见问题解答

## 特色功能

### ✨ 本教程的特点

- **全面系统** - 涵盖从入门到高级的所有知识点
- **循序渐进** - 按难度分级，适合不同阶段学习

- **中文友好** - 专为中文用户优化，易于理解
- **持续更新** - 跟随 Git 最新版本和最佳实践
- **开源共建** - 欢迎社区贡献和反馈

1. **循序渐进** - 按照学习路径一步步学习，打好基础
2. **动手实践** - 每学完一章就动手练习，加深理解
3. **完成练习** - 认真完成每章的练习题，巩固知识
4. **查阅参考** - 遇到问题时使用搜索功能快速查找
5. **分享交流** - 在 GitHub 上提问和分享学习心得
